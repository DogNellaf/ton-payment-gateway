# TON Payment Gateway Bot

> [🇬🇧 English](README.md) | 🇷🇺 Русский

Telegram-бот для приёма платежей в TON через инвойсы. Пользователь запрашивает инвойс на нужную сумму, получает ссылку Tonkeeper с уникальным комментарием, а бот автоматически подтверждает платёж, отслеживая транзакции на кошельке мерчанта.

> Только приём платежей: шлюз никогда не хранит приватные ключи и не подписывает транзакции. Ему нужны только **публичный** адрес кошелька мерчанта и API-ключ для чтения.

## Возможности

- 🧾 **Создание инвойсов** — запрос любой суммы в TON через встроенное меню.
- 🔗 **Диплинки Tonkeeper** — сумма и комментарий уже заполнены.
- 👁️ **Наблюдатель on-chain** — фоновая задача опрашивает кошелёк и сопоставляет входящие переводы с инвойсами по уникальному комментарию.
- 🔔 **Автоматическое подтверждение** — покупатель получает уведомление в момент зачисления средств.
- ⏳ **Жизненный цикл инвойса** — `pending → paid → expired`, с автоматическим истечением срока.
- 💰 **Целочисленные суммы** — хранятся в нанотонах (`BIGINT`), без чисел с плавающей точкой.

## Стек технологий

| Слой | Технология |
|---|---|
| Среда выполнения | Python 3.11, asyncio |

| База данных | PostgreSQL через asyncpg |
| Blockchain API | toncenter v2 HTTP API (aiohttp) |
| Тестирование | pytest |

## Требования

- Python 3.11+
- PostgreSQL

## Установка

```bash
# Установить зависимости
pip install -r requirements.txt

# Создать базу данных Postgres
createdb ton_gateway

# Настроить окружение
cp .env.example .env
# Отредактировать .env: BOT_TOKEN, MERCHANT_WALLET, DATABASE_URL, ...

# Запустить бот
python -m app
```

Схема базы данных создаётся автоматически при первом запуске.

> **Совет:** сначала попробуйте в тестовой сети — укажите `TON_API_BASE_URL=https://testnet.toncenter.com` и `TONKEEPER_TRANSFER_URL=https://test.tonkeeper.com/transfer`.

## Переменные окружения

| Переменная | Обязательна | По умолчанию | Описание |
|---|:---:|---|---|
| `BOT_TOKEN` | ✅ | — | Токен бота от @BotFather |
| `DATABASE_URL` | ✅ | — | DSN для PostgreSQL |
| `MERCHANT_WALLET` | ✅ | — | Публичный адрес кошелька для приёма |
| `TON_API_BASE_URL` | | `https://toncenter.com` | Базовый URL toncenter |
| `TON_API_KEY` | | — | API-ключ toncenter (повышенные лимиты) |
| `TONKEEPER_TRANSFER_URL` | | `https://app.tonkeeper.com/transfer` | Базовый URL диплинков для кнопок оплаты |
| `INVOICE_TTL_MINUTES` | | `30` | Время жизни инвойса |
| `POLL_INTERVAL_SECONDS` | | `20` | Интервал опроса кошелька |
| `MIN_AMOUNT_TON` | | `0.01` | Минимальная сумма инвойса |
| `MAX_AMOUNT_TON` | | `100` | Максимальная сумма инвойса |

## Запуск тестов

```bash
pip install -r requirements-dev.txt
pytest
```

Юнит-тесты покрывают арифметику сумм, формирование платёжных ссылок и парсинг транзакций — без сети и базы данных.

## Структура проекта

```
app/
  __init__.py
  __main__.py      # python -m app
  config.py        # настройки из окружения (.env)
  ton.py           # арифметика сумм, платёжные ссылки, клиент toncenter
  db.py            # репозиторий asyncpg, параметризованные запросы
  watcher.py       # фоновая задача подтверждения инвойсов
  bot.py           # обработчики Telegram + конфигурация приложения
  schema.sql       # схема БД (нанотоны как BIGINT)
tests/
  test_ton.py      # юнит-тесты чистых функций
.env.example
requirements.txt
```

## Безопасность

- Без приватных ключей: сервис работает **только на приём**.
- Секреты хранятся в `.env` (добавлен в `.gitignore`), ничего чувствительного не коммитится.
- Все SQL-запросы параметризованы через asyncpg — никакой интерполяции строк.
- Суммы обрабатываются как целые нанотоны, без потери точности.
